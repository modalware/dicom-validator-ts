# Security Policy

## Supported Versions

| Version | Supported          |
|---------|--------------------|
| 0.x     | :white_check_mark: |

## Reporting a Vulnerability

If you discover a security vulnerability in dicom-validator-ts, please report it responsibly.

### Preferred Method: GitHub Security Advisories

1. Go to the [Security Advisories](https://github.com/your-org/dicom-validator-ts/security/advisories) page.
2. Click "Report a vulnerability".
3. Provide a clear description of the issue, steps to reproduce, and potential impact.

### Alternative: Email

If you prefer email, send your report to: security@example.com

Please include:
- A description of the vulnerability
- Steps to reproduce the issue
- The potential impact
- Any suggested fixes (optional)

## Response Timeline

- **Acknowledgment**: Within 48 hours of receiving the report.
- **Initial assessment**: Within 5 business days.
- **Fix and disclosure**: We aim to release a fix within 30 days of confirming the vulnerability. We will coordinate disclosure timing with the reporter.

## Scope

This policy applies to the dicom-validator-ts npm package and its source code. It does not cover third-party dependencies — please report those to their respective maintainers.

## Recognition

We appreciate responsible disclosure and will credit reporters in the release notes (unless they prefer to remain anonymous).
